MCSA 70-410 Lab #4 - Install a GUI Utility Server

Oct 25, 2015 · 3 minute read · Comments

For the exam, it’s important to know how to convert between Core, Minimal Installation, and Full installation. We’re going to do some of that today. Our first subject will be a utility server. We can install all of the management tools on this one to manage the other servers.

The cool thing is that once we get this server running, we won’t need to start console sessions to any server we have already set up.

Looking back on the other lessons, complete the following steps. Try to do as much as possible from memory.

  • Create a server named Util1
  • Rename
  • Set timezone
  • Set a static IP
  • Set the DNS servers, pointing to both DC1 and DC2
  • Reboot

We haven’t gone over how to join the domain yet when you aren’t promoting to a DC. There are two main ways, the old and the new (PowerShell) way. I’m going with the new one.

# Method 1:
netdom join Util1 /domain:Home.local /userd:administrator /passwordd:**********


# Method 2:
Add-Computer -ComputerName Util1 -DomainName Home.local

# You can also specify the -NewName parameter to rename the computer and join the domain in one go.

If you have problems with this, especially if it mentions that the domain could not be contacted, remember that any problem you have is DNS. It’s always DNS.

If you have an issue, start with nslookup and ping, then go from there.

Convert Server Core to Full Install

Like I said before, this is one you will definitely be quizzed on.

Server core is really small and it doesn’t store the GUI binaries “just in case”. That means we need to provide them from the DVD. Mount that ISO again.

Get-VM Util1 | Add-VMDvdDrive -Path E:\Lab\iso\2012R2.ISO

It probably mounted on D:\, but you can check. 

Get-PSDrive

There is an install.wim file located at D:\sources\install.wim that has all of the information we need. Most Windows install disks will have a couple of editions (like Standard and Datacenter) in two flavors (Core and Full). We need to know which edition we’re running.

Get-WindowsEdition -Online

Edition : ServerDatacenter
# So I'm running Datacenter edition

And now we need to know the index of Datacenter edition in the install.wim file. 

Get-WindowsImage -ImagePath D:\sources\install.wim


ImageIndex       : 1
ImageName        : Windows Server 2012 R2 SERVERSTANDARDCORE
ImageDescription : Windows Server 2012 R2 SERVERSTANDARDCORE
ImageSize        : 7,460,086,466 bytes

ImageIndex       : 2
ImageName        : Windows Server 2012 R2 SERVERSTANDARD
ImageDescription : Windows Server 2012 R2 SERVERSTANDARD
ImageSize        : 13,068,096,479 bytes

ImageIndex       : 3
ImageName        : Windows Server 2012 R2 SERVERDATACENTERCORE
ImageDescription : Windows Server 2012 R2 SERVERDATACENTERCORE
ImageSize        : 7,457,614,120 bytes

ImageIndex       : 4
ImageName        : Windows Server 2012 R2 SERVERDATACENTER
ImageDescription : Windows Server 2012 R2 SERVERDATACENTER
ImageSize        : 13,067,663,813 bytes

So I need to target Index4, Datacenter full.

Install-WindowsFeature Server-Gui-Mgmt-Infra, Server-Gui-Shell -Restart -Source wim:D:\sources\install.wim:4
# The :4 at the end is the index

If the server was originally a GUI server and you were converting back, you would only have to run the second command and leave off the -Source parameter.

It will take a while, but once the server reboots you’re running in full GUI mode. Make sure to sign in as Home\Administrator.

Add both DCs to server manager and take a look. Our domain is coming along.

Finally, install the management tools. It’s simple.

Install-WindowsFeature RSAT

This installs all available management tools so that we can manage any other server from this one. Try it! You can connect to AD, DNS, etc.

That’s it for today. Next time we’ll… I’m not sure yet. But we have plenty to do!
